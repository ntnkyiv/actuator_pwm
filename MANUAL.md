## 1. Опис конструкції актуатора

### Загальна архітектура

Пристрій являє собою двоосьовий антенний ротатор під керуванням мікроконтролера **XIAO ESP32-S3**. Система забезпечує позиціонування антени по двох осях:

- **Азимут (горизонтальне обертання)** — кроковий двигун з редуктором
- **Кут місця (нахил/elevation)** — лінійний актуатор

Орієнтація в просторі вимірюється сенсором **BNO08x** (IMU з вбудованою MOTCAL-калібровкою), що дає абсолютні значення азимуту (yaw), тангажу (pitch) та крену (roll).

Зв'язок із зовнішньою системою керування здійснюється через **Ethernet** (міст CH9120 serial-to-Ethernet) або **WiFi** (вбудований веб-інтерфейс + WebSocket).

### Основні вузли

| Вузол | Компонент | Опис |
|-------|-----------|------|
| Мікроконтролер | XIAO ESP32-S3 | 240 МГц, 8 МБ Flash, WiFi/BT |
| Сенсор орієнтації | BNO08x (Adafruit) | 9-DoF IMU, I2C, MOTCAL |
| Привід азимуту | Кроковий двигун + драйвер STEP/DIR | AccelStepper, мікрокроковий |
| Привід нахилу | Лінійний актуатор | H-міст, PWM 25 кГц, 8-біт |
| Ethernet міст | CH9120 | UART ↔ TCP/UDP, 100 Мбіт |

### Параметри драйвера лінійного актуатора

- Частота PWM: **25 кГц** (безшумний режим)
- Роздільна здатність: **8 біт** (0–255)
- Режим гальмування (Brake): обидва входи HIGH
- Режим холостого ходу (Sleep): обидва входи LOW

### Параметри кроковика (значення за замовчуванням)

| Параметр | Значення | Команда |
|----------|----------|---------|
| Максимальна швидкість | 5000 кроків/с | `maxspeed` |
| Прискорення | 5000 кроків/с² | `acceleration` |
| Мінімальна тривалість імпульсу | 20 мкс | `pulsewidth` |
| Мікрокрок | 32 | `microstep` |
| Коефіцієнт редуктора | 9.7 | `reductor` |
| Крок двигуна | 1.8° | `stepsize` |
| **Кроків на градус** | **≈ 172.4** | розрахується автоматично |

> Формула: `кроків/градус = microstep × reductor / stepsize`

## 2. Рекомендації по встановленню

### Механічне монтування

- Встановлюйте пристрій так, щоб **вісь Z сенсора BNO08x була спрямована вертикально вгору** (Z = Up). Орієнтація осей: X = Схід, Y = Північ.
- Уникайте розміщення поряд із трансформаторами, моторами постійного струму та іншими джерелами магнітного поля — це погіршує точність компаса.
- Кабелі живлення до лінійного актуатора прокладайте окремо від сигнальних і I2C ліній.

### Електричне підключення

Пристрій живиться через **пасивний PoE 24 В** по кабелю Ethernet. Напруга 24 В подається безпосередньо на плату разом із даними.

> **Увага!** Пасивний PoE не має автоматичного узгодження — переконайтеся, що PoE-інжектор налаштований на **24 В** перед підключенням кабелю. Подача стандартного IEEE 802.3af/at (48 В) виведе пристрій з ладу.

- **Живлення мікроконтролера (3.3 В / 5 В):** формується з 24 В вбудованим DC-DC перетворювачем на платі.
- **Живлення кроковика і лінійного актуатора:** 24 В з PoE подаються безпосередньо на драйвери.
- Пін EN кроковика підтягнутий до HIGH (двигун вимкнений) за замовчуванням і переходить в LOW після ініціалізації. **Перед підключенням переконайтеся, що напрямок обертання відповідає зростанню азимуту за годинниковою стрілкою.**

### Перша ввімкнення

1. Підключіть кабель Ethernet від PoE-інжектора (24 В пасивний) до пристрою.
2. Протягом перших 5–15 секунд відбувається ініціалізація WiFi та BNO08x. Якщо сенсор не знайдено — система продовжує роботу без нього, але команди `azimuth`, `pry`, `pitch`, `compassmode`, `calibrate` будуть повертати помилку.

## 3. Початок роботи

### Підключення по WiFi

При першому ввімкненні (або якщо мережа STA недоступна) пристрій створює власну точку доступу:

| Параметр | Значення |
|----------|----------|
| SSID | `CompassActuator` |
| Пароль | `12345678` |
| IP-адреса | `192.168.4.1` |

1. Підключіться до мережі `CompassActuator` з будь-якого пристрою.
2. Відкрийте браузер та перейдіть за адресою `http://192.168.4.1`.
3. Для підключення до вашого роутера перейдіть до розділу **WiFi Setup** (посилання внизу сторінки), введіть SSID та пароль і натисніть **CONNECT**. Пристрій перезавантажиться і підключиться до мережі. IP-адресу дивіться у веб-інтерфейсі або через DHCP-таблицю роутера.

### Підключення по Ethernet (CH9120)

Підключіть кабель Ethernet до порту CH9120. За замовчуванням CH9120 налаштований на **TCP-сервер** (порт задається при конфігурації). Детальніше — у розділі [6. Налаштування CH9120](#6-налаштування-ch9120).

### Перевірка статусу

Надішліть команду для перевірки стану системи:

```json
{"cmd":"actuator_status"}
```

У відповідь отримаєте:

```json
{
  "cmd": "actuator_status",
  "uptime_ms": 12345,
  "compass_found": true,
  "yaw": 123.4,
  "pitch": 15.2,
  "roll": 0.5,
  "bno_fail_count": 0,
  "bno_last_ok_ms": 12100,
  "bno_stale_sec": 0,
  "i2c_0x4A": 0,
  "i2c_0x4B": 2,
  "bno_recovery_stage": 0,
  "stepper_pos": 0,
  "stepper_dist": 0
}
```

> `i2c_0x4A = 0` означає що сенсор відповідає на адресі 0x4A. `bno_recovery_stage = 0` — нормальна робота.

### Версія прошивки

```json
{"cmd":"version"}
```

Відповідь:
```json
{"cmd":"version","fw_version":"1.0.0","build_date":"Mar 25 2026 12:00:00","chip_id":"aabbccdd"}
```

## 4. Оновлення прошивки

Прошивка оновлюється через Ethernet (OTA over TCP) за допомогою Python-скрипта `ota_update.py`. WiFi OTA не підтримується.

### Підготовка

1. Встановіть Python 3 (якщо не встановлено).
2. Скопіюйте `actuator_pwm.ino.bin` в ту ж директорію, де знаходиться `ota_update.py`.

### Налаштування скрипта

Відкрийте `ota_update.py` та встановіть IP-адресу пристрою:

```python
ESP32_IP = "192.168.38.173"   # ← змініть на реальну IP вашого пристрою
PORT = 5000
FILENAME = "actuator_pwm.ino.bin"
```

### Запуск оновлення

```bash
python ota_update.py
```

Приклад виводу:

```
Розмір: 854320 байт
MD5: a1b2c3d4e5f6...
Команду з MD5 надіслано. Очікування...
Прогрес: 100.0%
Готово!
```

Після завершення пристрій автоматично перезавантажується і запускає нову прошивку.

### Важливо

- Під час OTA-оновлення **всі інші функції заблоковані** (головний цикл не виконується).
- Якщо передача перервана або MD5 не збіглась — пристрій залишається на старій прошивці і перезавантажується.
- Передача виконується блоками по **1024 байти** з паузою 10 мс між блоками для стабільності CH9120.

## 5. Опис веб-інтерфейсу

Веб-інтерфейс доступний за IP-адресою пристрою в браузері. Зв'язок відбувається через **WebSocket** (`ws://<ip>/ws`), оновлення даних компаса — **4 рази на секунду**.

### Головна сторінка (`/`)

**Рядок стану** (вгорі):
- **IP** — поточна IP-адреса пристрою
- **Compass** — стан сенсора BNO08x (з'являється "OK" зеленим після першого успішного читання)

**Блок даних сенсора:**

| Поле | Опис |
|------|------|
| AZIMUTH | Поточний азимут (yaw), градуси 0–360 |
| PITCH | Поточний кут нахилу, градуси |
| ROLL | Поточний крен, градуси |

**Кроковий двигун (Stepper Motor):**

- **Rotate (+/-)** — відносний поворот. Введіть кількість градусів (додатне = за годинниковою, від'ємне = проти). Натисніть кнопку.
- **Go to Azimuth** — абсолютне позиціонування за компасом. Введіть азимут 0–359°, натисніть кнопку. Потребує підключеного сенсора BNO08x.
- **STOP MOTOR** — негайна зупинка кроковика.

**Калібрування BNO08x:**

- Вкажіть кількість циклів (1–10) і натисніть **КАЛІБРУВАТИ**.
- З'явиться підтвердження про небезпеку руху. Після підтвердження антена виконає автоматичні рухи по обох осях для калібровки магнітометра (MOTCAL).
- Статус відображається під кнопкою: "Виконується..." → "Завершено ✓"

**Лінійний актуатор (Linear Actuator):**

| Елемент | Опис |
|---------|------|
| Кнопки 1–5 | Вибір рівня швидкості (1 = мінімум, 5 = максимум). Поточний рівень підсвічується. |
| Go to Pitch | Автоматичний рух до заданого кута pitch (потребує BNO08x). Поле ±90°. |
| EXTEND | Висунути актуатор (з обраною швидкістю) |
| RETRACT | Втягнути актуатор (з обраною швидкістю) |
| BRAKE | Активне гальмування (обидва виходи HIGH) |
| SLEEP | Вимкнення (обидва виходи LOW) |

> Після натискання EXTEND або RETRACT спрацює **авто-гальмування** через налаштований час (за замовчуванням 20 секунд).

**Посилання WiFi Setup** (внизу) — перехід до `/wifi` для зміни мережевих налаштувань.

### Сторінка WiFi Setup (`/wifi`)

Форма з двома полями: **SSID** та **Password**. Після відправки — пристрій зберігає дані в NVS і перезавантажується для підключення до нової мережі.

## 6. Налаштування CH9120

CH9120 — це апаратний міст між UART та Ethernet (TCP/UDP). Завдяки йому пристрій доступний через мережевий кабель без WiFi.

### Протокол UART між ESP32 і CH9120

| Параметр | Значення |
|----------|----------|
| Швидкість | 115200 бод |
| Формат | 8N1 |
| Буфер RX | 2048 байт |

### Вхід у режим налаштування CH9120

CH9120 налаштовується через власний утиліту **CH9120CFG** (від виробника WCH).

**Для конфігуратора CH9120CFG (Windows):**

1. Запустіть `CH9120NetModuleConfig.exe`.
2. Оберіть мережевий адаптер та натисніть кнопку **Search**.
3. Зробіть подвійний клік на знайденому адаптері CH9120.

### Рекомендовані налаштування

| Параметр | Рекомендоване значення |
|----------|------------------------|
| Режим роботи | TCP Server |
| Локальний порт | 5000 (для OTA); або будь-який (для команд) |
| IP-адреса (статична) | Наприклад, `192.168.1.100` |
| Маска підмережі | `255.255.255.0` |
| Шлюз | IP вашого роутера |
| Швидкість UART | **115200** бод |
| Формат даних | 8N1 |
| Режим передачі | Transparent (прозора передача) |

> **Важливо:** швидкість UART на CH9120 має збігатися зі швидкістю в прошивці — 115200 бод.

### Перезавантаження CH9120

Після збереження налаштувань CH9120 потрібно перезавантажити. ESP32 автоматично скидає CH9120 при кожному старті. 

### Режим TCP Server

У режимі TCP Server CH9120 чекає підключення від клієнта. Кожен байт, що прийшов з Ethernet, прозоро передається в UART до ESP32, і навпаки. Таким чином JSON-команди надсилаються безпосередньо в TCP-сокет.

Один момент підключення — один TCP-клієнт (CH9120 не підтримує декілька одночасних з'єднань у базовій конфігурації).

## 7. Робота з Packet Sender

**Packet Sender** — безкоштовна програма для відправки мережевих пакетів (TCP/UDP). Використовується для тестування та ручного керування пристроєм через Ethernet.

Завантажити: https://packetsender.com

### Налаштування підключення

1. Запустіть Packet Sender.
2. В полі **Address** вкажіть IP-адресу CH9120.
3. В полі **Port** вкажіть порт TCP-сервера CH9120 (наприклад, `5000`).
4. Оберіть тип пакету: **TCP**.
5. У налаштуваннях (Settings → Persistent TCP) увімкніть **Persistent TCP Connection** — це дозволить відправляти декілька команд без перевстановлення з'єднання.

### Формат команди

- В поле **ASCII** введіть JSON-команду.
- **Обов'язково** додайте символ нового рядка `\n` в кінці — прошивка зчитує рядки до `\n`.
- В Packet Sender `\n` вводиться як `\n` у режимі HEX або як новий рядок у режимі ASCII.

### Приклади команд

**Перевірка статусу:**
```
{"cmd":"actuator_status"}\n
```

**Повернути на 90° за годинниковою:**
```
{"cmd":"degree","value":90}\n
```

**Повернути в азимут 180°:**
```
{"cmd":"azimuth","value":180}\n
```

**Висунути лінійний актуатор на рівні 3:**
```
{"cmd":"lsetspeed","value":3}\n
```

**Зупинити кроковик:**
```
{"cmd":"stop","value":0}\n
```

### Отримання відповідей

Packet Sender відображає відповіді пристрою у вкладці **Traffic** або у нижньому рядку. Кожна команда повертає JSON-рядок із підтвердженням або даними.

### Типовий сеанс роботи

1. Встановіть TCP-з'єднання з пристроєм.
2. Надішліть `{"cmd":"actuator_status"}` — перевірте, що компас знайдено (`"compass_found":true`).
3. Надішліть `{"cmd":"azimuth","value":0}` — антена повернеться на північ.
4. Надішліть `{"cmd":"pitch","value":-5}` — лінійний актуатор підніме антену до 5°.
5. Надішліть `{"cmd":"lbrake"}` — зупинити актуатор вручну.

## 8. Детальний опис JSON команд

Протокол: **JSON рядки через UART1 / TCP (CH9120)**, термінатор — символ нового рядка (`\n`).

Формат запиту: `{"cmd":"назва_команди","value":значення}`
Формат відповіді: той самий JSON з доданими полями або зміненим `value`.

Якщо `"value": null` (або поле `value` відсутнє) — команда працює як **читання** (повертає поточне значення з NVS).
Якщо `"value"` вказане — команда виконує **запис/дію**.

---

### 8.1 Кроковий двигун

#### `maxspeed` — максимальна швидкість

```json
{"cmd":"maxspeed","value":5000}
```
Встановлює максимальну швидкість у кроках/с. Зберігається в NVS.

```json
{"cmd":"maxspeed","value":null}
```
Повертає поточне збережене значення.

| Поле | Тип | Опис |
|------|-----|------|
| value | int | Кроків/с. Рекомендований діапазон: 500–10000 |

---

#### `acceleration` — прискорення

```json
{"cmd":"acceleration","value":5000}
```
Встановлює прискорення у кроках/с². Зберігається в NVS.

```json
{"cmd":"acceleration","value":null}
```
Повертає поточне збережене значення.

---

#### `pulsewidth` — мінімальна тривалість імпульсу

```json
{"cmd":"pulsewidth","value":20}
```
Мінімальна тривалість STEP-імпульсу в мікросекундах. Зберігається в NVS. За замовчуванням: 20 мкс.

---

#### `microstep` — мікрокрок

```json
{"cmd":"microstep","value":32}
```
Коефіцієнт мікрокроку драйвера (1, 2, 4, 8, 16, 32...). Зберігається в NVS. Після зміни автоматично перераховується `stepper_ratio`.

---

#### `reductor` — коефіцієнт редуктора

```json
{"cmd":"reductor","value":9.7}
```
Передаточне відношення редуктора. Зберігається в NVS. Після зміни перераховується `stepper_ratio`.

---

#### `stepsize` — крок двигуна

```json
{"cmd":"stepsize","value":1.8}
```
Кутовий крок двигуна в градусах (зазвичай 1.8° або 0.9°). Зберігається в NVS. Після зміни перераховується `stepper_ratio`.

> **Формула stepper_ratio:** `microstep × reductor / stepsize`
> При значеннях за замовчуванням: `32 × 9.7 / 1.8 ≈ 172.4 кроків/°`

---

#### `stepper` — увімкнення/вимкнення двигуна

```json
{"cmd":"stepper","value":1}
```
`value: 1` — увімкнути (пін EN → LOW).
`value: 0` — вимкнути (пін EN → HIGH, двигун знеструмлений, вал вільний).

Відповідь:
```json
{"cmd":"stepper","value":1,"status":"enabled"}
```

---

#### `setcurrent` — встановити поточну позицію

```json
{"cmd":"setcurrent","value":0}
```
Встановлює поточну позицію AccelStepper (у кроках) без руху. Використовується для обнулення після механічного встановлення в референсне положення.

---

#### `absolute` — рух до абсолютної позиції (в кроках)

```json
{"cmd":"absolute","value":17240}
```
Рух до абсолютної позиції у кроках (тут ≈ 100° при ratio=172.4). Використовується для низькорівневого керування.

---

#### `relative` — відносний рух (в кроках)

```json
{"cmd":"relative","value":1724}
```
Рух на задану кількість кроків відносно поточної позиції. Від'ємне значення — зворотний напрямок.

---

#### `degree` — відносний рух (в градусах)

```json
{"cmd":"degree","value":90}
```
Рух на задану кількість градусів відносно поточної позиції. Від'ємне значення — зворотний напрямок.

---

#### `stop` — зупинка кроковика

```json
{"cmd":"stop","value":0}
```
Зупиняє кроковик (з уповільненням за профілем AccelStepper).

---

#### `azimuth` — рух до абсолютного азимуту (за компасом)

```json
{"cmd":"azimuth","value":270}
```
Розраховує найкоротший шлях від поточного yaw до цільового азимуту і виконує відповідний відносний рух кроковика. Потребує підключеного BNO08x.

```json
{"cmd":"azimuth","value":null}
```
Повертає поточний азимут за компасом (`currentYaw`).

Відповідь при русі:
```json
{"cmd":"azimuth","value":270.0}
```

Якщо компас не знайдено:
```json
{"cmd":"azimuth","error":"Компас не підключено"}
```

---

#### `compassmode` — режим постійного стеження за азимутом

```json
{"cmd":"compassmode","value":180}
```
Вмикає режим автоматичного стеження: кожні 100 мс контролер обчислює відхилення від заданого азимуту та виконує корекцію рухом кроковика. Потребує BNO08x.

```json
{"cmd":"compassmode","value":null}
```
Вимикає режим стеження.

### 8.2 Сенсор BNO08x

#### `pry` — поточні кути орієнтації

```json
{"cmd":"pry","value":null}
```

Відповідь:
```json
{
  "cmd": "pry",
  "pitch": 15.3,
  "roll": -2.1,
  "yaw": 247.8,
  "temp": 0
}
```

| Поле | Опис |
|------|------|
| yaw | Азимут 0–360° (0 = Північ, 90 = Схід) |
| pitch | Кут нахилу вперед/назад, ° |
| roll | Крен ліворуч/праворуч, ° |
| temp | Температура (поле зарезервовано, завжди 0) |

---

#### `calibrate` — калібрування магнітометра (MOTCAL)

```json
{"cmd":"calibrate","value":2}
```
Запускає процедуру калібрування MOTCAL на `value` циклів (мінімум 1). Кожен цикл: підняти антену → поворот –180° → опустити → поворот +360° → підняти → поворот –180° → опустити.

Відповідь при старті:
```json
{"cmd":"calibrate","status":"started","cycles":2}
```

Відповідь після завершення:
```json
{"cmd":"calibrate","status":"done"}
```

> **Увага!** Під час калібрування антена рухається механічно. Переконайтеся, що кабелі вільні та механізму нічого не заважає. Процедура блокує сеанс на весь час виконання.

### 8.3 Лінійний актуатор

#### `lextend` — висунути актуатор на максимальній швидкості

```json
{"cmd":"lextend"}
```
Висуває актуатор з PWM = 255. Після команди запускається таймер авто-гальмування.

---

#### `lretract` — втягнути актуатор на максимальній швидкості

```json
{"cmd":"lretract"}
```
Втягує актуатор з PWM = 255. Після команди запускається таймер авто-гальмування.

---

#### `lbrake` — активне гальмування

```json
{"cmd":"lbrake"}
```
Миттєво зупиняє актуатор у режимі гальмування (обидва виходи HIGH). Скасовує pitch-режим і таймер авто-гальмування.

---

#### `lsleep` — зупинка (вільний хід)

```json
{"cmd":"lsleep"}
```
Вимикає обидва виходи (LOW). Вал актуатора у вільному стані. Скасовує pitch-режим і таймер авто-гальмування.

---

#### `lsetspeed` — рух із заданим рівнем швидкості

```json
{"cmd":"lsetspeed","value":3}
```
Виконує рух із вказаним рівнем швидкості. Значення 1–5: висунути; –1..(–5): втягнути.

| Рівень | Напрямок | PWM (при minSpeed=0) |
|--------|----------|----------------------|
| 1 | Висунути | 0 |
| 2 | Висунути | ~64 |
| 3 | Висунути | ~127 |
| 4 | Висунути | ~191 |
| 5 | Висунути | 255 |
| -1..-5 | Втягнути | (аналогічно) |

> Якщо встановлено `linearMinSpeed` > 0, то рівень 1 відповідає `linearMinSpeed`, а не 0.

Відповідь:
```json
{"cmd":"lsetspeed","value":3,"pwm":127}
```

---

#### `pitch` — рух до заданого кута нахилу

```json
{"cmd":"pitch","value":30.0}
```
Вмикає режим автоматичного позиціонування за pitch-кутом. Актуатор рухається зі швидкістю рівня 2 до досягнення цілі (±0.2°) або виявлення механічного упору (немає зміни pitch понад 2 секунди).

```json
{"cmd":"pitch","value":null}
```
Повертає поточний стан:
```json
{
  "cmd": "pitch",
  "current_pitch": 15.3,
  "target_pitch": 30.0,
  "pitch_mode": true
}
```

---

#### `ltime` — таймер авто-гальмування

```json
{"cmd":"ltime","value":30000}
```
Встановлює час авто-гальмування у мілісекундах. Мінімум: 1000 мс, максимум: 300000 мс (5 хв). Зберігається в NVS.

```json
{"cmd":"ltime","value":null}
```
Повертає поточне значення.

За замовчуванням: **20000 мс (20 секунд)**.

---

#### `lminspeed` — мінімальна ненульова швидкість

```json
{"cmd":"lminspeed","value":80}
```
Встановлює мінімальне значення PWM, що подається на актуатор при будь-якому ненульовому русі. Корисно, якщо актуатор не рухається при малих значеннях PWM. Діапазон: 0–255. `0` = вимкнено. Зберігається в NVS.

```json
{"cmd":"lminspeed","value":null}
```
Повертає поточне значення.

## Додаток A. Коди помилок

| Поле відповіді | Значення | Причина |
|----------------|----------|---------|
| `"error":"json_parse"` | Помилка парсингу | Некоректний JSON або неповний рядок |
| `"error":"unknown_cmd"` | Невідома команда | Команда не існує |
| `"error":"Компас не підключено"` | BNO08x недоступний | Сенсор не знайдено при ініціалізації |

## Додаток B. Відновлення після збоїв

**BNO08x перестав відповідати:**
- Через 2 секунди без відповіді — автоматичний повторний `enableReport` (stage 1)
- Через 30 секунд — повторна ініціалізація `begin_I2C` (stage 2)
- При зависанні I2C шини — апаратне відновлення (10 імпульсів SCL)
- Якщо нічого не допомогло — надішліть `{"cmd":"restart"}`

**Кроковик не рухається:**
- Перевірте `stepper_ratio` > 0 командою `{"cmd":"actuator_status"}`
- Перевірте `{"cmd":"stepper","value":1}` — двигун має бути увімкнений
- Перевірте живлення драйвера

**Лінійний актуатор зупиняється сам по собі:**
- Спрацьовує авто-гальмування. Збільшіть час: `{"cmd":"ltime","value":60000}`
